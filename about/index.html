Editorial Experiment Note

Researcher’s Workbench

이 블로그는 AI 코딩 도구 리뷰가 아니라, 에이전트와 함께 일하는 개발 환경을 설계하고 검증하는 기록입니다.

Mission & Purpose

hangokudao는 성공한 결과만 모으는 포트폴리오가 아니라, AI 에이전트와 함께 일할 때 생기는 판단, 실패, 검증, 작업 규칙을 공개 가능한 형태로 남기는 기술 기록실입니다.

모든 글은 정보 밀도, 재현 가능성, 내부 정보 제거, 다음 작업에 재사용 가능한 규칙을 우선합니다.

Core Areas of Investigation

01 / Workflow

Agent Operating Rules

AGENTS.md, CLAUDE.md, 작업 지시와 검증 계약을 프로젝트 운영 방식으로 정리합니다.

02 / Validation

Failure to Rule

빌드 실패, 막힌 지점, 잘못된 가정을 다음 실행 규칙으로 바꾸는 과정을 기록합니다.

03 / Publishing

Draft Guard

초안, 민감 정보, 내부 경로를 분리하고 공개 가능한 글만 노출합니다.

Editorial Standards

  • 검증 로그는 명령과 결과를 함께 남깁니다.
  • 비공개 대화 원문, 내부 경로, 개인정보는 공개 전에 제거합니다.
  • 도구 찬양보다 실패 원인과 다음 규칙을 먼저 씁니다.
